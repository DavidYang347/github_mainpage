# 🤝 贡献指南

## 📋 贡献方式

### 1. 报告问题 (Issues)

如果你发现了问题或有改进建议：

1. 检查是否已有类似的 Issue
2. 如果没有，创建一个新的 Issue
3. 清楚地描述问题或建议
4. 如果可能，提供截图或示例

**Issue 模板**:
```markdown
**问题描述**
简要描述问题

**复现步骤**
1. 步骤 1
2. 步骤 2
3. ...

**期望行为**
你期望发生什么

**实际行为**
实际发生了什么

**环境信息**
- 浏览器：
- 操作系统：
- 其他相关信息：

**截图**
如果适用，添加截图帮助说明问题
```

---

### 2. 提交新模板

想要贡献新的模板风格？太棒了！

#### 要求
- ✅ 模板必须是完整可用的
- ✅ 包含详细的注释说明
- ✅ 遵循项目的命名规范
- ✅ 提供模板的预览说明

#### 步骤
1. Fork 本仓库
2. 在 `templates/` 目录创建新文件，命名为 `style-X-名称.md`
3. 在 `examples/preview.md` 中添加新模板的介绍
4. 更新主 `README.md`，在模板列表中添加新模板
5. 提交 Pull Request

#### 模板文件头部注释示例
```markdown
<!--
模板名称: [风格名称]
作者: [你的名字]
难度: ⭐⭐⭐
适合人群: [目标用户]
特点:
- 特点1
- 特点2
- 特点3
-->
```

## 🔀 Pull Request 流程

### 1. Fork 和 Clone

```bash
# Fork 本仓库（在 GitHub 网页上点击 Fork）

# Clone 你的 Fork
git clone https://github.com/你的用户名/github_mainpage.git
cd github_mainpage
```

### 2. 创建分支

```bash
# 创建并切换到新分支
git checkout -b feature/你的功能名称

# 或者修复 bug
git checkout -b fix/问题描述
```

### 3. 进行修改

- 遵循项目的代码风格
- 保持 Markdown 格式一致
- 添加必要的注释
- 测试你的修改

### 4. 提交更改

```bash
# 添加修改的文件
git add .

# 提交（使用清晰的提交信息）
git commit -m "添加：新的模板风格 - 渐变主题"

# 或者
git commit -m "修复：简约风格的徽章链接错误"

# 或者
git commit -m "改进：快速开始指南的说明清晰度"
```

#### 提交信息规范

使用以下前缀：
- `添加：` - 新增功能或文件
- `修复：` - 修复 bug
- `改进：` - 改进现有功能
- `更新：` - 更新文档或依赖
- `删除：` - 删除文件或功能
- `重构：` - 代码重构

### 5. 推送到 GitHub

```bash
git push origin feature/你的功能名称
```

### 6. 创建 Pull Request

1. 访问你 Fork 的仓库页面
2. 点击 "Pull Request" 按钮
3. 填写 PR 模板（见下方）
4. 提交 Pull Request

## 🎨 风格指南

### Markdown 格式

#### 标题层级
```markdown
# 一级标题（每个文件只有一个）
## 二级标题
### 三级标题
#### 四级标题
```

#### 列表
```markdown
<!-- 无序列表 -->
- 项目 1
- 项目 2
  - 子项目 2.1
  - 子项目 2.2

<!-- 有序列表 -->
1. 第一步
2. 第二步
3. 第三步
```

#### 代码块
```markdown
<!-- 内联代码 -->
使用 `代码` 格式

<!-- 代码块 -->
\`\`\`markdown
代码内容
\`\`\`
```

#### 链接
```markdown
<!-- 内部链接 -->
[文件名](路径/到/文件.md)

<!-- 外部链接 -->
[显示文本](https://url.com)
```

#### 表格
```markdown
| 左对齐 | 居中 | 右对齐 |
|:---|:---:|---:|
| 内容 | 内容 | 内容 |
```

### 文件命名

- 使用小写字母和连字符
- 模板文件: `style-数字-名称.md`
- 文档文件: `描述性名称.md`
- 示例: `customization-example.md`

### 注释规范

在模板中添加注释帮助用户：

```markdown
<!-- 这是用户需要修改的部分 -->
![Stats](https://github-readme-stats.vercel.app/api?username=你的用户名&...)
<!-- 将 '你的用户名' 替换为你的 GitHub 用户名 -->
```

## ✅ 质量标准

### 对于新模板

- [ ] 包含完整的个人信息部分
- [ ] 技术栈展示清晰
- [ ] 集成至少 2 种 GitHub 统计
- [ ] 包含社交媒体链接部分
- [ ] 所有占位符都用明确的标记（如 `你的用户名`）

- [ ] 格式统一
- [ ] 信息准确
- [ ] 链接有效

- [ ] 工具仍在维护中
