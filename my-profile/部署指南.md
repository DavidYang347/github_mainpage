# 🚀 GitHub 个人主页部署指南

这份指南将帮助你在 5 分钟内将数据驱动风格的 GitHub 主页部署到你的个人资料中。

---

## 📋 部署前准备

### 必须完成的修改

在部署之前，你**必须**将 `README.md` 文件中的以下占位符替换为你的真实信息：

#### 1. 基础信息（共 9 处）

| 占位符 | 需要替换为 | 示例 |
|:---|:---|:---|
| `你的名字` | 你的真实姓名或昵称 | `张三` |
| `你的GitHub用户名` | 你的 GitHub 用户名 | `zhangsan` |
| `your.email@example.com` | 你的邮箱 | `zhangsan@example.com` |
| `你的GitHub用户ID` | 你的 GitHub 用户 ID | `12345678` |
| `你的用户名` | 社交媒体用户名 | `zhangsan` |
| `yourblog.com` | 你的博客地址 | `https://zhangsan.dev` |

**重要**: `你的GitHub用户名` 在文件中出现多次，需要全部替换！

#### 2. 如何查找你的 GitHub 用户 ID？

访问：`https://api.github.com/users/你的用户名`

在返回的 JSON 中找到 `"id"` 字段，那就是你的用户 ID。

例如：
```json
{
  "login": "zhangsan",
  "id": 12345678,
  ...
}
```

## 🔧 快速替换指南

### 使用文本编辑器批量替换

#### 方法 1: 使用 VS Code

1. 打开 `README.md` 文件
2. 按 `Ctrl+H`（Windows/Linux）或 `Cmd+H`（Mac）打开查找替换
3. 依次替换以下内容：

```
查找: 你的GitHub用户名
替换为: zhangsan （替换为你的实际用户名）
点击 "Replace All"

查找: 你的名字
替换为: 张三 （替换为你的实际名字）
点击 "Replace All"

查找: your.email@example.com
替换为: zhangsan@example.com （替换为你的邮箱）
点击 "Replace All"

查找: 你的GitHub用户ID
替换为: 12345678 （替换为你的实际 ID）
点击 "Replace All"

查找: 你的用户名
替换为: zhangsan （替换为你的社交媒体用户名）
点击 "Replace All"

查找: yourblog.com
替换为: https://zhangsan.dev （替换为你的博客地址）
点击 "Replace All"
```

#### 方法 2: 使用命令行（Linux/Mac）

```bash
# 进入 my-profile 目录
cd my-profile

# 批量替换（请修改为你的实际信息）
sed -i 's/你的GitHub用户名/zhangsan/g' README.md
sed -i 's/你的名字/张三/g' README.md
sed -i 's/your\.email@example\.com/zhangsan@example.com/g' README.md
sed -i 's/你的GitHub用户ID/12345678/g' README.md
sed -i 's/你的用户名/zhangsan/g' README.md
sed -i 's/yourblog\.com/https:\/\/zhangsan.dev/g' README.md
```

#### 方法 3: 使用 PowerShell（Windows）

```powershell
# 进入 my-profile 目录
cd my-profile

# 批量替换（请修改为你的实际信息）
(Get-Content README.md) -replace '你的GitHub用户名', 'zhangsan' | Set-Content README.md
(Get-Content README.md) -replace '你的名字', '张三' | Set-Content README.md
(Get-Content README.md) -replace 'your\.email@example\.com', 'zhangsan@example.com' | Set-Content README.md
(Get-Content README.md) -replace '你的GitHub用户ID', '12345678' | Set-Content README.md
(Get-Content README.md) -replace '你的用户名', 'zhangsan' | Set-Content README.md
(Get-Content README.md) -replace 'yourblog\.com', 'https://zhangsan.dev' | Set-Content README.md
```

## 📤 部署到 GitHub

### 步骤 1: 创建特殊仓库

1. **登录 GitHub**
   - 访问 https://github.com
   - 登录你的账号

2. **创建新仓库**
   - 点击右上角的 `+` 号
   - 选择 `New repository`

3. **设置仓库名称**（非常重要！）
   - **仓库名称必须与你的 GitHub 用户名完全相同**
   - 例如：如果你的用户名是 `zhangsan`，仓库名就是 `zhangsan`
   - ✅ 正确示例：用户名 `zhangsan` → 仓库名 `zhangsan`
   - ❌ 错误示例：用户名 `zhangsan` → 仓库名 `my-profile`

4. **仓库设置**
   - ✅ 选择 `Public`（必须是公开的）
   - ✅ 勾选 `Add a README file`
   - 点击 `Create repository`

5. **确认成功**
   - 如果仓库名正确，GitHub 会显示：
   ```
   ✨ You found a secret! 
   zhangsan/zhangsan is a ✨ special ✨ repository. 
   Its README.md will appear on your profile!
   ```

### 步骤 2: 上传你的 README

#### 方法 A: 通过 GitHub 网页（推荐，最简单）

1. **打开刚创建的仓库**
   - 访问 `https://github.com/你的用户名/你的用户名`

2. **编辑 README.md**
   - 点击 `README.md` 文件
   - 点击右上角的 ✏️ 编辑按钮

3. **替换内容**
   - 删除默认内容
   - 复制你修改好的 `my-profile/README.md` 的全部内容
   - 粘贴到编辑器中

4. **提交更改**
   - 滚动到页面底部
   - 在 "Commit changes" 输入提交信息，例如：`更新个人主页`
   - 点击 `Commit changes` 按钮

#### 方法 B: 通过 Git 命令行

```bash
# 1. Clone 你的仓库
git clone https://github.com/你的用户名/你的用户名.git
cd 你的用户名

# 2. 复制修改好的 README
cp ../my-profile/README.md ./

# 3. 提交更改
git add README.md
git commit -m "更新个人主页 - 数据驱动风格"
git push origin main
```

#### 方法 C: 使用 GitHub Desktop

1. **打开 GitHub Desktop**
2. **Clone 仓库**
   - File → Clone Repository
   - 选择你刚创建的仓库
   - 选择本地保存位置
3. **复制文件**
   - 将修改好的 `README.md` 复制到仓库目录
4. **提交更改**
   - 在 GitHub Desktop 中会看到变更
   - 输入提交信息：`更新个人主页`
   - 点击 `Commit to main`
   - 点击 `Push origin`

## 🔧 常见问题排查

### Q1: README 没有显示在个人主页上？

**检查清单**：
- [ ] 仓库名称是否与用户名完全相同？（大小写也要一致）
- [ ] 仓库是否设置为 Public（公开）？
- [ ] 文件名是否为 `README.md`？（注意大小写）
- [ ] 是否已经等待 1-2 分钟让 GitHub 刷新？

### Q2: 统计图片不显示？

**可能原因和解决方法**：

1. **用户名未替换**
   - 检查所有 `你的GitHub用户名` 是否都已替换
   
2. **服务暂时不可用**
   - 等待几分钟后刷新
   - 尝试访问图片链接查看是否有错误信息

3. **URL 格式错误**
   - 确保用户名中没有多余空格
   - 检查 URL 是否完整

### Q3: 某些特殊字符显示不正常？

**解决方法**：
- 确保文件使用 UTF-8 编码
- 在 VS Code 中，右下角可以看到文件编码
- 如果不是 UTF-8，点击编码名称，选择 "Save with Encoding"，选择 UTF-8

### Q4: Mermaid 思维导图不显示？

**原因**：
- GitHub 的 Mermaid 支持可能需要一段时间加载
- 某些浏览器可能不支持

**解决方法**：
- 刷新页面
- 如果长期不显示，可以考虑用图片替代

### Q5: 如何更新内容？

**方法**：
1. 直接在 GitHub 网页上编辑 README.md
2. 或者 clone 到本地修改后 push

```bash
git clone https://github.com/你的用户名/你的用户名.git
cd 你的用户名
# 编辑 README.md
git add README.md
git commit -m "更新内容"
git push
```
