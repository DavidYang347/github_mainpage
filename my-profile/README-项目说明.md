# 📦 GitHub 个人主页部署包

这是一个完整的 GitHub 个人主页部署包，采用**数据驱动风格**，包含了所有必要的文件和详细的部署说明。

## 📁 文件说明

### 核心文件

#### 1. `README.md`
- **作用**: GitHub 个人主页的主文件，将显示在你的 GitHub 个人资料页面
- **状态**: ⚠️ 需要修改（包含占位符）
- **下一步**: 替换所有占位符为你的真实信息

#### 2. `部署指南.md`
- **作用**: 详细的部署步骤说明文档
- **内容包括**:
  - 部署前的准备工作
  - 快速替换指南
  - 三种部署方法（网页、命令行、GitHub Desktop）
  - 常见问题排查
  - 进阶优化技巧
- **建议**: 部署前仔细阅读

#### 3. `个人信息配置.txt`
- **作用**: 个人信息配置模板和批量替换命令
- **内容包括**:
  - 基础信息填写区域
  - 社交媒体配置
  - 自动生成的替换命令
  - PowerShell 和 Bash 脚本模板
  - 检查清单
- **使用方法**: 按照模板填写后，使用生成的命令批量替换

### 自动化脚本

#### 4. `update-profile.ps1`
- **作用**: Windows PowerShell 自动更新脚本
- **功能**:
  - 自动替换所有占位符
  - 创建备份文件
  - 显示更新统计
  - 可选自动打开编辑器
- **使用方法**:
  ```powershell
  # 1. 编辑脚本，在"配置区域"填写你的信息
  # 2. 运行脚本
  .\update-profile.ps1
  ```

#### 5. `update-profile.sh`
- **作用**: Linux/Mac Bash 自动更新脚本
- **功能**: 与 PowerShell 脚本相同
- **使用方法**:
  ```bash
  # 1. 编辑脚本，在"配置区域"填写你的信息
  # 2. 添加执行权限
  chmod +x update-profile.sh
  # 3. 运行脚本
  ./update-profile.sh
  ```

#### 6. `README-项目说明.md`
- **作用**: 本文件，项目整体说明文档
- **内容**: 文件清单、使用流程、注意事项等

## 🚀 快速开始

### 方法一：使用自动化脚本（推荐）⭐

**Windows 用户：**

1. 打开 `update-profile.ps1`
2. 找到"配置区域"（第18-40行左右）
3. 填写你的真实信息：
   ```powershell
   $config = @{
       userName = "你的GitHub用户名"
       displayName = "你的名字"
       email = "你的邮箱"
       # ... 其他信息
   }
   ```
4. 保存文件
5. 右键点击文件，选择"使用 PowerShell 运行"
   - 或在 PowerShell 中执行：`.\update-profile.ps1`
6. 按提示操作

**Linux/Mac 用户：**

1. 打开 `update-profile.sh`
2. 找到"配置区域"（第15-25行左右）
3. 填写你的真实信息：
   ```bash
   USER_NAME="你的GitHub用户名"
   DISPLAY_NAME="你的名字"
   EMAIL="你的邮箱"
   # ... 其他信息
   ```
4. 保存文件
5. 在终端中执行：
   ```bash
   chmod +x update-profile.sh
   ./update-profile.sh
   ```
6. 按提示操作

### 方法二：手动替换

1. 打开 `个人信息配置.txt`
2. 填写你的信息
3. 使用文本编辑器（如 VS Code）的查找替换功能
4. 按照配置文件中的说明依次替换

### 方法三：使用编辑器查找替换

1. 用支持批量替换的编辑器打开 `README.md`（推荐 VS Code）
2. 使用查找替换功能（Ctrl+H 或 Cmd+H）
3. 依次替换以下内容：
   - `你的GitHub用户名` → 你的实际用户名
   - `你的名字` → 你的实际名字
   - `your.email@example.com` → 你的邮箱
   - 等等...

## ✅ 部署流程

### 步骤 1: 准备文件 ✏️

- [ ] 使用上述任一方法替换占位符
- [ ] 检查所有信息是否正确
- [ ] 保存文件

### 步骤 2: 创建 GitHub 仓库 🗂️

1. 登录 GitHub
2. 创建新仓库
3. **仓库名称必须与你的 GitHub 用户名完全相同**
4. 设置为 Public（公开）
5. 勾选 "Add a README file"
6. 创建仓库

### 步骤 3: 上传 README 📤

**方法 A - 网页编辑（最简单）：**
1. 打开刚创建的仓库
2. 点击 README.md 文件
3. 点击编辑按钮（✏️）
4. 复制粘贴你修改好的内容
5. 提交更改

**方法 B - Git 命令行：**
```bash
git clone https://github.com/你的用户名/你的用户名.git
cd 你的用户名
cp ../my-profile/README.md ./
git add README.md
git commit -m "更新个人主页"
git push
```

**方法 C - GitHub Desktop：**
1. Clone 仓库到本地
2. 复制修改好的 README.md
3. 提交并推送

### 步骤 4: 查看效果 🎉

访问：`https://github.com/你的用户名`

等待 1-2 分钟让 GitHub 刷新缓存，然后就能看到精美的个人主页了！

## 📝 必须替换的占位符清单

在部署之前，确保替换以下所有占位符：

### 高优先级（必须替换）

| 占位符 | 出现次数 | 位置 | 影响 |
|:---|:---:|:---|:---|
| `你的GitHub用户名` | ~10次 | 统计图表URL | 不替换会导致统计无法显示 |
| `你的名字` | 1次 | 顶部标题 | 显示错误的名字 |
| `your.email@example.com` | 2次 | 邮箱链接 | 邮件地址错误 |
| `你的GitHub用户ID` | 1次 | 底部统计 | 统计可能不准确 |

### 中优先级（建议替换）

| 占位符 | 说明 |
|:---|:---|
| `你的用户名` | 社交媒体链接中的用户名 |
| `yourblog.com` | 个人博客地址 |
| `全栈开发工程师` | 职位描述 |
| `中国` | 所在地 |

### 低优先级（可选替换）

- 年度统计数据
- 技能熟练度百分比
- 工作经历内容
- 年度目标
- 认证徽章

## ⚠️ 注意事项

### 关键要点

1. **仓库名称必须正确**
   - 必须与 GitHub 用户名完全相同
   - 大小写敏感
   - 错误示例：用户名 `ZhangSan`，仓库名 `zhangsan` ❌
   - 正确示例：用户名 `ZhangSan`，仓库名 `ZhangSan` ✅

2. **仓库必须公开**
   - 只有 Public 仓库的 README 才会显示在个人主页
   - Private 仓库不会生效

3. **文件名必须正确**
   - 必须是 `README.md`
   - 大小写敏感（全大写的 README，小写的 .md）

4. **GitHub 用户名必须准确**
   - 统计图表的 URL 中包含用户名
   - 拼写错误会导致图表无法加载
   - 多次出现，必须全部正确

5. **文件编码**
   - 使用 UTF-8 编码保存
   - 避免使用带 BOM 的 UTF-8

6. **首次部署**
   - 等待 1-2 分钟让 GitHub 刷新
   - 如果没有显示，尝试清除浏览器缓存

### 常见错误

❌ **错误 1**: 仓库名与用户名不一致
- 后果：README 不会显示在个人主页

❌ **错误 2**: 忘记替换 `你的GitHub用户名`
- 后果：统计图表显示其他人的数据或无法显示

❌ **错误 3**: 仓库设置为 Private
- 后果：README 不会公开显示

❌ **错误 4**: 文件名拼写错误（readme.md, README.MD）
- 后果：无法被识别为 Profile README
